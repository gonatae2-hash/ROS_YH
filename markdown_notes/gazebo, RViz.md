# 🤖 Gazebo vs RViz 비교 & 연관관계

> ROS에서 가장 많이 쓰이는 두 도구의 차이점과 관계 정리

---

## 📌 한줄 정의

| 도구 | 정의 |
|------|------|
| **Gazebo** | 로봇을 가상 세계에서 **시뮬레이션**하는 3D 환경 |
| **RViz** | ROS 데이터를 **실시간으로 시각화**하는 모니터링 도구 |

---

## ⚔️ 핵심 차이점

```
Gazebo                              RViz
─────────────────────────────────────────────────
🌍 가상 세계를 만들어 줌              👀 데이터를 눈으로 보여줌
🤖 로봇을 실제처럼 동작시킴           📊 로봇이 보내는 데이터 표시
⚙️ 물리 법칙 적용 O (중력, 충돌)      ⚙️ 물리 법칙 적용 X
🎮 "로봇을 움직이는 곳"              🖥️ "로봇이 뭘 보는지 보는 곳"
💡 입력 역할                         💡 출력 역할
```

---

## 🔍 상세 비교

| 항목 | Gazebo | RViz |
|------|--------|------|
| 목적 | 시뮬레이션 | 시각화 |
| 물리 엔진 | O (중력, 마찰, 충돌) | X |
| 실제 로봇 없이 사용 | O | O |
| 실제 로봇과 함께 사용 | X | O ✅ |
| 센서 데이터 생성 | O (가짜 센서) | X (받아서 표시만) |
| 주요 용도 | 개발/테스트 | 디버깅/모니터링 |

---

## 📡 각각 시각화/시뮬레이션 가능한 것

### Gazebo
- 로봇 물리적 움직임 (바퀴, 관절, 팔)
- 가상 센서 데이터 생성 (라이다, 카메라, IMU)
- 지형/장애물/중력 등 환경
- 충돌 및 마찰 시뮬레이션

### RViz
| 데이터 | 표시 방식 |
|------|------|
| 로봇 모델 | 3D 모델로 관절/링크 표시 |
| 라이다 | 장애물까지 거리를 점(Point Cloud)으로 표시 |
| 카메라 | 로봇 시점 영상 실시간 표시 |
| 이동 경로 | 선(Path)으로 표시 |
| TF 좌표계 | 각 부품 좌표를 화살표로 표시 |
| 목표 지점 | 마커로 표시 |

## 🔗 연관관계

```
┌─────────────────────────────────────────────┐
│                내 ROS 코드                   │
│         (timer_server.py 같은 노드)          │
└──────────────────┬──────────────────────────┘
                   │ 명령 전송
        ┌──────────▼──────────┐
        │       Gazebo         │
        │  (가상 로봇 동작)     │
        │  물리 법칙 적용       │
        └──────────┬──────────┘
                   │ 센서 데이터 발행 (Topic)
        ┌──────────▼──────────┐
        │        RViz          │
        │  (데이터 시각화)      │
        │  실시간 모니터링      │
        └─────────────────────┘
```

---

## 🛠️ 실제 개발 워크플로우

```
1단계: 코드 작성
    내 ROS 노드 작성 (Action, Service, Topic)

2단계: Gazebo에서 테스트
    실제 로봇 없이 가상으로 동작 확인
    실수해도 재시작하면 됨 💥→🔄

3단계: RViz로 모니터링
    센서 데이터, 경로, 좌표계 시각적으로 확인
    버그 위치 빠르게 파악

4단계: 실제 로봇 적용
    코드 그대로 실제 로봇에 올림
    RViz는 실제 로봇과도 계속 사용
```

## 💻 실행 명령어

```bash
# Gazebo 단독 실행
roslaunch gazebo_ros empty_world.launch

# RViz 단독 실행
rosrun rviz rviz

# 둘 다 한번에 실행 (launch 파일로)
roslaunch [패키지명] [launch파일명].launch
```

## ⭐ 핵심 요약

> **Gazebo** = 로봇을 위한 게임 엔진 🎮 → 시뮬레이션  
> **RViz** = 로봇을 위한 모니터 🖥️ → 시각화  
>
> Gazebo가 만든 데이터를 RViz가 보여주는 관계  
> 개발할 때 보통 **같이** 사용함
