## 1. 📄 URDF 기본 구조

> **Unified Robot Description Format**
> 로봇의 생김새, 관절, 물리 특성을 XML로 표현하는 파일

### 기본 골격
```xml
<?xml version="1.0"?>
<robot name="my_robot">

    <!-- 링크 정의 -->
    <link name="base_link">
        <visual/>       <!-- 눈에 보이는 모양 -->
        <collision/>    <!-- 충돌 판정 영역 -->
        <inertial/>     <!-- 질량/관성 (Gazebo용) -->
    </link>

    <!-- 조인트 정의 -->
    <joint name="joint1" type="revolute">
        <parent link="base_link"/>
        <child link="next_link"/>
        <origin xyz="0 0 0.1"/>
        <axis xyz="0 0 1"/>
        <limit lower="-3.14" upper="3.14"
               effort="300" velocity="0.1"/>
    </joint>

</robot>
```

## 2. 🔗 링크 (Link) 3가지 구성요소

### 👁️ visual - 눈에 보이는 외형
```xml
<visual>
    <geometry>
        <cylinder length="0.4" radius="0.04"/>
    </geometry>
    <origin rpy="0 0 0" xyz="0 0 0"/>
    <material name="red">
        <color rgba="1 0 0 1"/>   <!-- R G B 투명도 -->
    </material>
</visual>
```
> ✅ RViz에서 보이는 모양
> ✅ 물리 계산에 영향 없음

### 💥 collision - 충돌 판정 영역
```xml
<collision>
    <geometry>
        <cylinder length="0.42" radius="0.06"/>  <!-- visual보다 살짝 크게 -->
    </geometry>
    <origin rpy="0 0 0" xyz="0 0 0"/>
</collision>
```
> ✅ Gazebo에서 충돌 계산에 사용
> ✅ 게임의 "히트박스" 개념
> ✅ visual보다 조금 크게 설정하는 것이 관례

### ⚖️ inertial - 질량과 관성
```xml
<inertial>
    <mass value="1"/>                         <!-- 질량 (kg) -->
    <inertia ixx="1.0" ixy="0.0" ixz="0.0"
             iyy="1.0" iyz="0.0" izz="1.0"/> <!-- 관성 텐서 -->
</inertial>
```
> ✅ Gazebo 물리 시뮬레이션에서만 사용
> ✅ RViz에서는 영향 없음
> ⚠️ root link(base_link)에 inertial 있으면 KDL 경고 발생
> 💡 해결: dummy_link 추가

### 📦 geometry 모양 종류
```xml
<box size="0.1 0.1 0.1"/>                  <!-- 가로 세로 높이 -->
<cylinder length="0.4" radius="0.04"/>      <!-- 길이 반지름 -->
<sphere radius="0.05"/>                     <!-- 반지름 -->
<mesh filename="package://pkg/meshes/robot.stl"/>  <!-- 3D 파일 -->
```

## 3. 🔄 조인트 (Joint) 6가지 타입

| 타입 | 움직임 | 제한 | 주요 사용처 |
|------|--------|------|------------|
| `revolute` | 회전 | 있음 | 로봇 팔, pan/tilt |
| `continuous` | 무한 회전 | 없음 | 바퀴, 프로펠러 |
| `prismatic` | 직선 이동 | 있음 | 그리퍼, 리프트 |
| `fixed` | 고정 | - | 카메라 고정, dummy |
| `floating` | 6자유도 | 없음 | 드론 |
| `planar` | 평면 이동 | - | 평면 로봇 |

### 조인트 주요 태그
```xml
<joint name="pan_joint" type="revolute">
    <parent link="base_link"/>      <!-- 부모 링크 -->
    <child link="pan_link"/>        <!-- 자식 링크 -->
    <origin xyz="0 0 0.1"           <!-- 부모 기준 위치 -->
            rpy="0 0 0"/>           <!-- 부모 기준 회전 -->
    <axis xyz="0 0 1"/>             <!-- 회전 축 -->
    <limit lower="-3.14"            <!-- 최소 각도 (rad) -->
           upper="3.14"             <!-- 최대 각도 (rad) -->
           effort="300"             <!-- 최대 힘 (N) -->
           velocity="0.1"/>         <!-- 최대 속도 (rad/s) -->
    <dynamics damping="50"          <!-- 감쇠 -->
              friction="1"/>        <!-- 마찰 -->
</joint>
```

## 5. ✍️ xacro - URDF를 편하게 작성하기

> URDF는 반복이 많고 길어짐 → xacro로 변수/함수 사용 가능

### 변수 선언 및 사용
```xml
<!-- 변수 선언 -->
<xacro:property name="wheel_radius" value="0.05"/>
<xacro:property name="wheel_length" value="0.1"/>

<!-- 변수 사용 -->
<cylinder radius="${wheel_radius}" length="${wheel_length}"/>

<!-- 수식 사용 -->
<cylinder radius="${wheel_radius * 2}" length="${wheel_length + 0.02}"/>
```

### 매크로 (함수처럼 사용)
```xml
<!-- 매크로 정의 -->
<xacro:macro name="wheel" params="name x y">
    <link name="${name}">
        <visual>
            <geometry>
                <cylinder length="${wheel_length}" radius="${wheel_radius}"/>
            </geometry>
        </visual>
    </link>
    <joint name="${name}_joint" type="continuous">
        <origin xyz="${x} ${y} 0"/>
        <parent link="base_link"/>
        <child link="${name}"/>
        <axis xyz="0 1 0"/>
    </joint>
</xacro:macro>

<!-- 매크로 사용 (함수 호출처럼) -->
<xacro:wheel name="left_wheel"  x="0" y="0.15"/>
<xacro:wheel name="right_wheel" x="0" y="-0.15"/>
```

### 파일 include
```xml
<xacro:include filename="$(find my_pkg)/urdf/materials.xacro"/>
```

### xacro → URDF 변환
```bash
rosrun xacro xacro robot.urdf.xacro > robot.urdf
```

## 6. 🔧 핵심 패키지 5가지

### 전체 흐름
```
xacro → URDF → kdl_parser → robot_state_publisher → RViz
                                    ↑
                     joint_state_publisher (관절값 공급)
                     joint_state_publisher_gui (슬라이더)
```

| 패키지 | 역할 |
|--------|------|
| `xacro` | URDF를 편하게 작성하는 매크로 도구 |
| `kdl_parser` | URDF → KDL 트리 변환 (내부적으로 자동) |
| `robot_state_publisher` | 관절값 + URDF → /tf 발행 |
| `joint_state_publisher` | 관절값 자동 발행 (테스트용) |
| `joint_state_publisher_gui` | 슬라이더 GUI로 관절 조작 |

> ⚠️ `joint_state_publisher`와 `joint_state_publisher_gui`는 **동시 실행 금지**
> 실제 로봇에서는 **모터 엔코더**가 joint_state_publisher 역할 대신함

## 7. 🚀 launch 파일 작성법

```xml
<launch>
    <!-- xacro → URDF 변환 후 파라미터 등록 -->
    <param name="robot_description"
           command="$(find xacro)/xacro
                    $(find my_pkg)/urdf/robot.urdf.xacro"/>

    <!-- 관절값 GUI 슬라이더 -->
    <node name="joint_state_publisher_gui"
          pkg="joint_state_publisher_gui"
          type="joint_state_publisher_gui"/>

    <!-- TF 계산 및 발행 -->
    <node name="robot_state_publisher"
          pkg="robot_state_publisher"
          type="robot_state_publisher"/>

    <!-- RViz 실행 -->
    <node name="rviz"
          pkg="rviz"
          type="rviz"
          args="-d $(find my_pkg)/urdf.rviz"/>
</launch>
```

## 9. 🐛 자주 나오는 에러 & 해결법

| 에러 | 원인 | 해결 |
|------|------|------|
| `material undefined` | material 미정의 | URDF 상단에 material 추가 |
| `Geometry tag contains no child` | geometry 태그 비어있음 | box/cylinder/sphere 추가 |
| `root link has inertia` | base_link에 inertial 있음 | dummy_link 추가 |
| `Inertial must have inertia element` | inertia 태그 누락 | `<inertia>` 태그 추가 |
| `허가 거부` | 실행 권한 없음 | `roslaunch`로 실행 |
| `cannot find executable xacro.py` | Noetic에서 이름 변경됨 | `xacro.py` → `xacro` |

## 12. 📐 rpy / xyz 이해하기

### xyz - 위치
```
xyz="0.1 0.2 0.3"
      ↑   ↑   ↑
      X   Y   Z  (미터 단위)
```

### rpy - 회전
```
rpy="0 1.57 0"
      ↑  ↑  ↑
    Roll Pitch Yaw  (라디안 단위)

1.57 rad ≈ 90도
3.14 rad ≈ 180도
```

### 라디안 ↔ 도 변환
| 라디안 | 도 |
|--------|-----|
| 0 | 0° |
| 1.57 | 90° |
| 3.14 | 180° |
| 6.28 | 360° |
